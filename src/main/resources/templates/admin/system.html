系统说明（答辩可用）

技术选型、功能模块与安全设计要点

技术栈

  • 后端：Java 21、Spring Boot 3、Spring MVC、Spring Data JPA、Spring Security、Bean Validation
  • 视图：Thymeleaf 服务端渲染，默认转义输出，降低 XSS 风险
  • 数据库：开发 H2；生产 MySQL（可通过环境变量切换数据源）
  • 接口：REST JSON（/api/**）与页面表单并存，便于对比「前后端分离 vs 传统 SSR」

功能模块

  • 认证：表单登录、注册、BCrypt 密码散列、基于角色的访问控制（USER / ADMIN）
  • 留言：主贴 + 一层回复、分页、点赞（仅主贴）、作者或管理员删除
  • 管理端：统计看板、用户全生命周期管理、留言检索与删除、CSV 导出
  • 个人中心：查看资料、修改密码（成功后要求重新登录）

安全与部署提示

生产环境务必修改默认管理员口令，限制数据库与 SSH 访问，启用 HTTPS；GitHub Actions 部署脚本中通过密钥注入环境变量，避免将数据库密码写入仓库。 公开接口中写操作需登录；静态资源与 /help 帮助页可匿名访问便于演示。
